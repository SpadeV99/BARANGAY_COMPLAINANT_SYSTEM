1. Introduction

The Barangay Complaint Management System is a web-based application designed for local government units (barangays) in the Philippines to efficiently manage and process community complaints and concerns. The system digitalizes the traditional manual complaint filing process, providing transparency and improving service delivery to residents.

1.1 Purpose

  • Streamline the complaint filing and tracking process
  • Provide a centralized database of complaints
  • Enable efficient assignment and resolution of community issues
  • Offer transparency to residents regarding the status of their complaints
  • Generate reports and analytics for barangay officials

1.2 Target Users

  • Barangay residents
  • Barangay staff and officials
  • Barangay administrators

2. System Architecture

2.1 Technology Stack

  • Backend: Python Flask framework
  • Database: MySQL with SQLAlchemy ORM
  • Frontend: HTML, CSS, JavaScript
  • UI Framework: Bootstrap 5
  • Icons: Font Awesome 6

  • Authentication: Flask-Login

5. Database Schema

5.1 User Model

class User(db.Model, UserMixin):
    id = db.Column(db.Integer, primary_key=True)
    username = db.Column(db.String(50), unique=True, nullable=False)
    email = db.Column(db.String(120), unique=True, nullable=False)
    password_hash = db.Column(db.String(255))
    first_name = db.Column(db.String(30), nullable=False)
    last_name = db.Column(db.String(30), nullable=False)
    contact_number = db.Column(db.String(15))
    address = db.Column(db.String(255))
    role = db.Column(db.String(20), default='resident')  # admin, staff, resident
    date_registered = db.Column(db.DateTime, default=datetime.utcnow)

5.2 Complaint Model

class Complaint(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    title = db.Column(db.String(100), nullable=False)
    description = db.Column(db.Text, nullable=False)
    date_filed = db.Column(db.DateTime, default=datetime.utcnow)
    status = db.Column(db.String(20), default='pending')  # pending, processing, resolved, dismissed
    resolution = db.Column(db.Text)
    resolution_date = db.Column(db.DateTime)
    
    # Foreign Keys
    complainant_id = db.Column(db.Integer, db.ForeignKey('user.id'), nullable=False)
    assigned_to = db.Column(db.Integer, db.ForeignKey('user.id'))
    
    # Relationships
    complainant = db.relationship('User', foreign_keys=[complainant_id], backref='filed_complaints')
    handler = db.relationship('User', foreign_keys=[assigned_to], backref='assigned_complaints')

6. Installation Guide

Note: The following instructions assume you have basic knowledge of command line operations and database management.

6.1 Prerequisites

  • Python 3.8+
  • MySQL/MariaDB
  • Pip (Python package manager)
  • Virtual environment (recommended)

6.2 Installation Steps

Step 1: Clone the repository
git clone <repository-url>
cd BARANGAY_COMPLAINANT_SYSTEM
Step 2: Create and activate virtual environment
python -m venv .venv
# On Windows:
.venv\Scripts\activate
# On macOS/Linux:
source .venv/bin/activate
Step 3: Install dependencies
pip install -r requirements.txt
Step 4: Configure environment variables

Create or modify the .env file:

SECRET_KEY=your_secret_key_replace_with_something_secure
DATABASE_URL=mysql+pymysql://root:@localhost/barangay_system
FLASK_APP=run.py
FLASK_ENV=development
Step 5: Create the database
mysql -u root -p
CREATE DATABASE barangay_system;
exit;
Step 6: Initialize the database
python init_db.py
Step 7: Run the application
python run.py
Step 8: Access the application

Open a web browser and navigate to http://localhost:5000

8. Technical Details

8.1 Authentication System

  • User authentication is handled by Flask-Login
  • Passwords are hashed using Werkzeug's security functions
  • Session management is handled by Flask's session system

8.2 Form Handling

  • Forms are created using Flask-WTF
  • Form validation includes both client-side and server-side validation
  • CSRF protection is enabled for all forms

8.3 Database Operations

  • Database operations are handled through SQLAlchemy ORM
  • Relationships between tables are maintained through foreign keys
  • Transactions ensure data integrity

8.4 Frontend Features

  • Responsive design using Bootstrap 5
  • Custom CSS with Philippine flag-themed colors
  • Interactive elements using JavaScript
  • Search functionality for tables
  • Modals for confirmations and quick actions

8.5 Template Structure

  • Base template (base.html) provides the common layout
  • Template inheritance for consistent design
  • Specialized templates for different sections (auth, complaints)
  • Flash messages for user notifications

9. Maintenance

9.1 Backup Procedures

Regularly backup the MySQL database:

mysqldump -u root -p barangay_system > backup_$(date +"%Y%m%d").sql

Store backups in a secure, offsite location.

9.2 Updating the System

  1. Pull the latest code from the repository
  2. Activate the virtual environment
  3. Install any new dependencies
  4. Run database migrations if needed
  5. Restart the application

9.3 Troubleshooting Common Issues

  • Verify database credentials in .env file
  • Ensure MySQL service is running
  • Check network connectivity

  • Check the application logs for error details
  • Verify Python and dependency versions
  • Ensure adequate system resources

  • Verify user credentials
  • Check user role assignments
  • Clear browser cache and cookies
